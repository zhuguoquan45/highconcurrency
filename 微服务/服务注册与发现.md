# 服务注册与发现

## 为什么需要服务注册发现

微服务部署后 IP 和端口动态变化，客户端不能硬编码地址。服务注册发现解决：
- 服务上线自动注册
- 服务下线自动摘除
- 客户端动态获取可用实例列表

---

## 核心流程

```
服务启动 → 注册中心（etcd/consul）← 服务发现 ← 客户端
服务下线 → 注销 / 心跳超时自动摘除
```

## etcd 实现

### 服务注册（带 TTL 心跳）

```go
import (
    clientv3 "go.etcd.io/etcd/client/v3"
    "go.etcd.io/etcd/client/v3/naming/endpoints"
)

func registerService(client *clientv3.Client, serviceName, addr string) error {
    // 创建租约（TTL=10s）
    lease, _ := client.Grant(context.Background(), 10)

    // 注册服务
    key := fmt.Sprintf("/services/%s/%s", serviceName, addr)
    _, err := client.Put(context.Background(), key, addr,
        clientv3.WithLease(lease.ID))

    // 自动续约（保持心跳）
    ch, _ := client.KeepAlive(context.Background(), lease.ID)
    go func() {
        for range ch {} // 消费续约响应
    }()

    return err
}
```

### 服务发现（Watch 变更）

```go
func discoverService(client *clientv3.Client, serviceName string) {
    prefix := "/services/" + serviceName + "/"

    // 获取当前实例列表
    resp, _ := client.Get(context.Background(), prefix,
        clientv3.WithPrefix())
    for _, kv := range resp.Kvs {
        fmt.Printf("instance: %s\n", kv.Value)
    }

    // Watch 变更
    watchCh := client.Watch(context.Background(), prefix,
        clientv3.WithPrefix())
    for event := range watchCh {
        for _, ev := range event.Events {
            switch ev.Type {
            case clientv3.EventTypePut:
                fmt.Printf("new instance: %s\n", ev.Kv.Value)
            case clientv3.EventTypeDelete:
                fmt.Printf("instance removed: %s\n", ev.Kv.Key)
            }
        }
    }
}
```

## consul 实现

```go
import "github.com/hashicorp/consul/api"

// 注册服务
func registerWithConsul(addr, port string) {
    config := api.DefaultConfig()
    client, _ := api.NewClient(config)

    registration := &api.AgentServiceRegistration{
        ID:      "order-service-1",
        Name:    "order-service",
        Address: addr,
        Port:    8080,
        Check: &api.AgentServiceCheck{
            HTTP:     fmt.Sprintf("http://%s:%s/health", addr, port),
            Interval: "10s",
            Timeout:  "3s",
        },
    }
    client.Agent().ServiceRegister(registration)
}

// 发现服务
func discoverWithConsul(serviceName string) []*api.ServiceEntry {
    client, _ := api.NewClient(api.DefaultConfig())
    services, _, _ := client.Health().Service(serviceName, "", true, nil)
    return services
}
```

## 负载均衡

```go
// 简单轮询
type RoundRobin struct {
    mu       sync.Mutex
    addrs    []string
    current  int
}

func (r *RoundRobin) Next() string {
    r.mu.Lock()
    defer r.mu.Unlock()
    addr := r.addrs[r.current%len(r.addrs)]
    r.current++
    return addr
}
```

常见策略：
- 轮询（Round Robin）：均匀分配
- 加权轮询：按服务器性能分配
- 随机：简单高效
- 一致性哈希：同一 key 路由到同一节点（适合有状态服务）

## etcd vs consul 对比

| 对比 | etcd | consul |
|------|------|--------|
| 一致性 | Raft | Raft |
| 健康检查 | 需自己实现（TTL） | 内置（HTTP/TCP/Script） |
| 服务发现 | 需自己实现 | 内置 |
| DNS 支持 | 无 | 有 |
| 适用场景 | K8s 配置存储 | 微服务注册发现 |

## 面试高频问题

**Q: 服务注册发现有哪些方案？**
- etcd：K8s 生态，强一致性
- consul：内置健康检查和服务发现，功能完整
- ZooKeeper：Java 生态，较重
- Nacos：阿里开源，支持配置中心+服务发现

**Q: 服务下线如何保证流量不丢失？**
1. 先从注册中心摘除（停止接收新请求）
2. 等待已有请求处理完（graceful shutdown）
3. 再停止进程

```go
// Go 优雅退出
quit := make(chan os.Signal, 1)
signal.Notify(quit, syscall.SIGTERM, syscall.SIGINT)
<-quit

// 1. 从注册中心注销
deregister()

// 2. 等待进行中的请求完成
ctx, cancel := context.WithTimeout(context.Background(), 30*time.Second)
defer cancel()
server.Shutdown(ctx)
```

**Q: 注册中心挂了怎么办？**
- 客户端本地缓存服务列表，注册中心恢复前继续使用缓存
- 注册中心本身做高可用（etcd 3 节点集群）
- 服务间直连作为降级方案

**Q: 心跳超时多久合适？**
- TTL 太短：网络抖动导致误摘除
- TTL 太长：服务宕机后长时间不摘除
- 通常 TTL=10-30s，心跳间隔 = TTL/3
