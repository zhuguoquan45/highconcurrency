# Go 高并发岗位数据库设计综合文档

---

## 1. 核心设计理念

在互联网高并发场景下，数据库设计需要兼顾：

1. **高性能**

   * 优化查询和索引
   * 热点数据缓存
   * 批量操作减少压力

2. **高可用**

   * 主从复制、读写分离
   * 自动故障切换

3. **高扩展性**

   * 分库分表
   * 水平扩展

4. **数据一致性**

   * 事务（ACID / TCC / Saga / MQ 异步事务）
   * 异常恢复机制

**总结理念**：索引 + 缓存 + 队列 + 分库分表 + 异步事务 + 高可用部署

---

## 2. 数据库选型

| 场景      | 类型      | 推荐                    | 特点             |
| ------- | ------- | --------------------- | -------------- |
| 核心业务    | 关系型     | MySQL / PostgreSQL    | 支持事务、复杂查询、强一致性 |
| 高频访问    | 内存型     | Redis                 | 热点缓存，高速读写，原子操作 |
| 日志 / 分析 | 列式 / 时序 | ClickHouse / InfluxDB | 大数据分析，不适合频繁写   |

> 初学者建议先用 MySQL + Redis，熟练后扩展到分析型数据库。

## 3. 表设计示例

### 3.1 用户表

```sql
CREATE TABLE users (
    user_id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(64) NOT NULL UNIQUE,
    password_hash CHAR(64) NOT NULL,
    status TINYINT DEFAULT 1,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

* **说明**：

  * 主键唯一
  * 用户名唯一索引保证注册唯一
  * 登录信息可缓存 Redis

---

### 3.2 订单表

```sql
CREATE TABLE orders (
    order_id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    product_id BIGINT NOT NULL,
    amount DECIMAL(10,2) NOT NULL,
    status TINYINT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_user_id(user_id),
    INDEX idx_status(status)
);
```

* **高并发优化**：

  * 分库分表：按 `user_id % N`
  * 队列异步写入，避免瞬间写爆数据库

---

### 3.3 产品表

```sql
CREATE TABLE products (
    product_id BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(128) NOT NULL,
    price DECIMAL(10,2) NOT NULL,
    stock INT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_name(name)
);
```

* **高频读优化**：

  * Redis 缓存库存和商品信息
  * 秒杀场景使用 Lua 脚本保证原子扣减

## 4. 索引设计原则

1. **为查询而建索引**：用于 `WHERE`、`JOIN`、`ORDER BY`、`GROUP BY`
2. **选择性高的列优先索引**
3. **组合索引遵循最左前缀原则**
4. **覆盖索引**：索引包含查询字段，减少回表
5. **避免低效索引**：函数计算、低选择性字段
6. **平衡读写开销**：索引越多，写入越慢

---

## 5. 高并发优化策略

1. **缓存设计**

   * Redis 热点缓存
   * 先查缓存 → 缓存没命中再查数据库
   * 写数据库后刷新缓存（异步/延迟）

2. **批量操作**

   * 插入多条记录用批量 SQL
   * 避免 N 次单条写入

3. **读写分离**

   * 主库处理写
   * 多从库处理读

4. **队列 + 异步处理**

   * 高频写入先入队列
   * 异步落库，避免瞬时压力峰值

5. **事务与一致性**

   * 单库事务：BEGIN / COMMIT / ROLLBACK
   * 分布式事务：TCC / Saga / MQ 异步事务

---

## 6. 分库分表设计

* **分库**：按业务线或用户 ID 拆库
* **分表**：按 ID 或时间拆表
* **查询路由**：

  * 中间件：ShardingSphere / MyCAT
  * Go 服务直接路由（简单逻辑）

> Tip：初学者先单库单表练手，再过渡到分库分表。

---

## 7. 高可用与运维

1. **主从复制 + 自动故障切换**

   * 主库挂掉 → 从库提升为主库
   * Go 服务使用连接池 + 读写分离

2. **连接池配置（Go）**

```go
db.SetMaxOpenConns(100)
db.SetMaxIdleConns(50)
```

3. **慢查询监控**

   * MySQL `slow_query_log`
   * Go 端记录慢 SQL，及时优化

## 8. 高并发场景示例

| 场景      | 优化策略               | Go 实践           |
| ------- | ------------------ | --------------- |
| 秒杀 / 抢购 | Redis 队列 + Lua 扣库存 | 避免瞬时 DB 压力      |

| 分布式事务   | TCC / Saga / MQ    | 保证最终一致性         |
