这个问题几乎是 **Go 面试必问 TOP10**。一句话先总结：

> **goroutine 是 Go 语言在用户态实现的轻量级线程，由 Go runtime 调度；线程是操作系统调度的执行单元。**

下面从几个关键维度拆开讲（面试非常加分）。

---

## 1. 创建和销毁成本

**线程（Thread）**

* 由 **操作系统创建**
* 需要 **系统调用**
* 创建成本高（几十微秒级）
* 默认栈空间大

Linux 线程默认栈：

```
8 MB
```

如果开 1 万线程：

```
8MB × 10000 = 80GB
```

基本直接爆内存。

---

**goroutine**

* 由 **Go runtime 创建**
* 不需要系统调用
* 创建成本极低（纳秒级）
* 初始栈非常小

Go 1.20+：

```
初始栈：2KB
```

而且 **栈会自动扩容和缩容**。

示例：

```go
for i := 0; i < 1000000; i++ {
    go func() {
        fmt.Println("hello")
    }()
}
```

Go 可以轻松运行 **百万级 goroutine**。

---

## 2. 调度方式

### 线程调度

线程由 **操作系统调度**。

调度模型：

```
OS Scheduler
```

特点：

* 抢占式调度
* 上下文切换成本高
* 内核态参与

线程切换过程：

```
保存寄存器
切换栈
切换地址空间
恢复寄存器
```

开销较大。

---

### goroutine 调度

goroutine 由 **Go runtime 调度**。

调度模型叫：

```
GMP模型
```

三个核心对象：

```
G = Goroutine
M = Machine（OS线程）
P = Processor（调度器）
```

关系：

```
G -> P -> M -> CPU
```

特点：

* 用户态调度
* 切换成本非常低
* runtime 控制

---

## 3. 栈内存

### 线程

线程栈：

```
固定大小
```

Linux：

```
8MB
```

如果递归深度不够：

```
Stack Overflow
```

---

### goroutine

goroutine 栈：

```
动态增长
```

初始：

```
2KB
```

需要更多空间：

```
2KB -> 4KB -> 8KB -> 16KB -> ...
```

自动扩容。

不用了会 **自动收缩**。

---

## 4. 数量级

线程：

```
通常几千就到极限
```

例如：

```
5000 ~ 10000
```

再多：

* 内存爆
* 调度爆
* 上下文切换爆

---

goroutine：

```
几十万 ~ 百万级
```

很多 Go 服务器：

```
100万 goroutine
```

都很常见。

---

## 5. 切换成本

线程切换：

```
内核态
```

成本：

```
几微秒
```

需要：

* 保存寄存器
* 系统调用
* 调度器决策

---

goroutine 切换：

```
用户态
```

成本：

```
纳秒级
```

只需要：

```
保存少量寄存器
切换栈
```

因此 Go 非常适合 **高并发 IO**。

---

## 6. 阻塞行为

### 线程

如果线程阻塞：

```
read()
sleep()
lock()
```

那么：

```
整个线程阻塞
```

OS 需要再调度别的线程。

---

### goroutine

如果 goroutine 阻塞：

```
channel
network IO
sleep
```

Go runtime 会：

```
挂起 goroutine
释放线程
调度其他 goroutine
```

线程不会被浪费。

---

## 7. 通信方式

线程：

通常使用：

```
mutex
condition variable
```

容易出现：

* 死锁
* 竞态条件

---

goroutine：

Go 提倡：

> **不要通过共享内存通信，而是通过通信共享内存**

典型方式：

```
channel
```

例子：

```go
ch := make(chan int)

go func() {
    ch <- 1
}()

v := <-ch
```

---

## 8. 总结对比（面试背这个）

| 对比项  | goroutine  | 线程    |
| ---- | ---------- | ----- |
| 调度   | Go runtime | 操作系统  |
| 创建成本 | 非常低        | 很高    |
| 初始栈  | 2KB        | 8MB   |
| 栈大小  | 动态增长       | 固定    |
| 切换成本 | 纳秒级        | 微秒级   |
| 调度方式 | 用户态        | 内核态   |
| 数量级  | 百万级        | 千级    |
| 通信   | channel    | mutex |

一句面试总结：

> **goroutine 是 Go runtime 管理的轻量级线程，具有更低的创建成本、更小的栈空间和用户态调度，因此可以支持百万级并发，而传统线程由操作系统调度，资源消耗较大。**
