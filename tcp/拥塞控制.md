TCP **拥塞控制（Congestion Control）**是 TCP 为了避免网络被流量压垮而设计的一套 **动态调节发送速率的算法**。

一句面试总结版：

> **TCP 通过慢启动、拥塞避免、快速重传和快速恢复四个机制，根据网络拥塞情况动态调整拥塞窗口（cwnd），从而控制发送速率。**

核心思想：

```text
网络好 → 加快发送
网络拥塞 → 降低发送
```

---

# 1 TCP 为什么需要拥塞控制

如果没有拥塞控制：

```text
发送方一直疯狂发包
↓
路由器队列被打满
↓
大量丢包
↓
网络崩溃（Congestion Collapse）
```

因此 TCP 需要根据网络状况控制发送速率。

TCP 主要依赖两个窗口：

| 窗口       | 含义          |
| -------- | ----------- |
| **rwnd** | 接收窗口（接收方能力） |
| **cwnd** | 拥塞窗口（网络能力）  |

真正发送窗口：

```text
send_window = min(rwnd, cwnd)
```

---

# 2 TCP 拥塞控制四大算法

TCP 拥塞控制有 **四个核心机制**：

```text
1 慢启动 (Slow Start)
2 拥塞避免 (Congestion Avoidance)
3 快速重传 (Fast Retransmit)
4 快速恢复 (Fast Recovery)
```

---

# 3 慢启动（Slow Start）

刚开始发送数据时：

```text
cwnd = 1 MSS
```

每收到一个 ACK：

```text
cwnd = cwnd * 2
```

即：

```text
指数增长
```

增长过程：

```text
1
2
4
8
16
32
```

直到达到 **慢启动阈值（ssthresh）**。

---

# 4 拥塞避免（Congestion Avoidance）

当：

```text
cwnd >= ssthresh
```

进入拥塞避免阶段。

增长方式变成：

```text
线性增长
```

即：

每个 RTT：

```text
cwnd = cwnd + 1 MSS
```

增长变慢：

```text
32
33
34
35
```

避免网络突然拥塞。

---

# 5 快速重传（Fast Retransmit）

TCP 通过 **重复 ACK** 判断丢包。

如果收到：

```text
3 个重复 ACK
```

说明：

```text
某个包丢了
```

立即重传：

```text
丢失的数据包
```

而不等待：

```text
RTO 超时
```

这样可以更快恢复。

---

# 6 快速恢复（Fast Recovery）

当检测到丢包：

TCP 会认为：

```text
网络出现拥塞
```

此时：

```text
ssthresh = cwnd / 2
cwnd = ssthresh
```

然后继续进入：

```text
拥塞避免
```

而不是重新慢启动。

# 8 超时 vs 重复ACK

TCP 有两种丢包检测方式：

| 情况          | 处理              |
| ----------- | --------------- |
| **RTO 超时**  | cwnd = 1（重新慢启动） |
| **3个重复ACK** | 快速重传 + 快速恢复     |

所以：

```text
RTO 认为网络严重拥塞
重复ACK 认为只是轻微丢包
```

# 9 常见 TCP 拥塞算法

现代 TCP 还有很多算法：

| 算法          | 说明         |
| ----------- | ---------- |
| **Reno**    | 经典 TCP     |
| **NewReno** | 改进恢复       |
| **CUBIC**   | Linux 默认算法 |
| **BBR**     | Google 算法  |

Linux 默认：

```text
CUBIC
```

Google：

```text
BBR
```

BBR 是基于 **带宽估计**，不是丢包。

# 10 面试标准回答（40秒）

如果面试官问：

**TCP 拥塞控制机制是什么？**

可以这样回答：

> TCP 拥塞控制通过动态调整拥塞窗口 cwnd 来控制发送速率，主要包括四个机制：慢启动、拥塞避免、快速重传和快速恢复。连接开始时 cwnd 从 1 MSS 开始指数增长，这是慢启动阶段；达到阈值 ssthresh 后进入拥塞避免阶段，窗口线性增长；如果收到 3 个重复 ACK，则触发快速重传并进入快速恢复，将 cwnd 减半；如果发生超时，则认为网络严重拥塞，cwnd 重置为 1 并重新慢启动。通过这些机制 TCP 可以根据网络情况动态调整发送速率，避免网络拥塞。
