## **一、基础命令 & 数据结构（题 1–8）**

**1. Redis 常用数据类型有哪些？适用场景？**

* **考点**：数据结构与用途
* **答案**：

  * **String**：计数器、缓存值
  * **Hash**：存储对象字段
  * **List**：队列、任务流
  * **Set**：无重复元素集合
  * **ZSet**：有序集合，排行榜
* **原理解析**：不同数据结构提供不同复杂度的操作，便于高并发操作优化
* **面试提示**：基础数据结构掌握情况

**2. Redis 支持哪些持久化方式？**

* **考点**：RDB 与 AOF
* **答案**：

  * **RDB**：快照持久化，定时保存，恢复快，但可能丢失数据
  * **AOF**：追加日志，每条命令写入文件，可配置 fsync，恢复慢但可靠
* **原理解析**：RDB 对 CPU 影响小，AOF 可提供高数据安全
* **面试提示**：了解 Redis 高可用与持久化

**3. Redis 单线程如何支撑高并发？**

* **考点**：单线程性能原理
* **答案**：

  * Redis 使用单线程 I/O 多路复用（epoll/kqueue）
  * 操作时间复杂度低（O(1) 或 O(logN)）
* **原理解析**：避免线程切换开销，高效利用 CPU
* **面试提示**：理解单线程原理下高并发

**4. Redis 命令复杂度如何影响性能？**

* **考点**：大 key 操作
* **答案**：

  * O(N) 或 O(logN) 的操作会阻塞主线程
  * 避免大 key 批量操作，如 KEYS、LRANGE 全量
* **原理解析**：单线程阻塞影响吞吐
* **面试提示**：高并发场景优化意识

**5. Redis pipeline 的作用？**

* **考点**：批量操作优化
* **答案**：

  * 将多条命令打包一次发送，减少网络 RTT
* **原理解析**：降低客户端与 Redis 交互延迟
* **面试提示**：优化吞吐量常用手段

**6. Redis Hash 如何在高并发下节省内存？**

* **考点**：内存优化
* **答案**：

  * 小 hash (<512字段) 使用 ziplist 编码
  * 大 hash 自动转换为 hashtable
* **原理解析**：压缩存储，提高缓存效率
* **面试提示**：面试官关注内存优化思路

**7. List 在高并发场景的应用？**

* **考点**：队列应用
* **答案**：

  * 队列/消息任务流，LPUSH + RPOP
  * 阻塞操作：BLPOP/BRPOP
* **原理解析**：阻塞队列可实现生产者-消费者模型
* **面试提示**：任务队列设计

**8. ZSet 的高并发应用场景？**

* **考点**：有序集合应用
* **答案**：排行榜、延迟队列、带权重排序
* **原理解析**：底层跳表结构，O(logN) 插入和删除
* **面试提示**：考察对高并发排序场景理解

## **二、缓存策略（题 9–15）**

**9. Redis 缓存穿透是什么？解决方案？**

* **考点**：缓存安全
* **答案**：

  * 穿透：查询不存在的 key，每次都打到 DB
  * 解决：

    * 布隆过滤器（BloomFilter）
    * 空对象缓存（带短 TTL）
* **原理解析**：减少 DB 压力，防止高并发穿透
* **面试提示**：缓存安全题

**10. 缓存击穿是什么？如何防止？**

* **考点**：热点 key 并发
* **答案**：

  * 击穿：热点 key 过期，大量请求穿透到 DB
  * 解决：互斥锁 / Lua 脚本 + 续期 / 预热缓存
* **原理解析**：限制同一 key 并发访问 DB
* **面试提示**：高并发优化场景

**11. 缓存雪崩是什么？如何解决？**

* **考点**：缓存一致性
* **答案**：

  * 大量 key 同时过期 → DB 压力大
  * 解决：TTL 随机化、预热、分批过期
* **原理解析**：避免瞬时高并发请求打到 DB
* **面试提示**：缓存安全与性能题

**12. Cache Aside 模式如何保证一致性？**

* **考点**：缓存更新策略
* **答案**：

  * 读：先读缓存 → 缓存不存在读 DB → 写缓存
  * 写：先写 DB → 删除/更新缓存
* **原理解析**：异步缓存更新保证最终一致性
* **面试提示**：分布式缓存设计

**13. 如何设计热点数据缓存？**

* **考点**：高并发热点处理
* **答案**：

  * 热点数据长 TTL 或永不过期
  * 可配合 LFU/LRU 策略
* **原理解析**：减少热点 key 重复加载 DB
* **面试提示**：缓存优化能力

**14. Redis LRU 策略如何工作？**

* **考点**：淘汰策略
* **答案**：

  * 使用访问时间近似算法淘汰最近最少使用的 key
  * 可配置 maxmemory-policy：volatile-lru, allkeys-lru
* **原理解析**：减少内存占用，保证高并发性能
* **面试提示**：缓存管理理解

**15. 缓存穿透、击穿、雪崩三者区别？**

* **考点**：缓存安全概念
* **答案**：

  * 穿透：查询不存在 key → DB
  * 击穿：热点 key 过期 → DB
  * 雪崩：大量 key 同时过期 → DB 压力大
* **原理解析**：核心在请求量对 DB 压力
* **面试提示**：理论基础题

## **三、分布式锁（题 16–22）**

**16. Redis 分布式锁原理？**

* **考点**：高并发互斥
* **答案**：

  * 使用 `SET key value NX PX timeout`
  * NX：不存在才设置
  * PX：过期时间
* **原理解析**：保证同一时刻只有一个客户端获得锁
* **面试提示**：分布式一致性考察

**17. RedLock 是什么？**

* **考点**：高可用分布式锁
* **答案**：

  * 多个 Redis 节点加锁 → 多数节点成功即持锁
* **原理解析**：保证单点故障不丢失锁
* **面试提示**：分布式锁高级题

**18. 分布式锁如何避免死锁？**

* **考点**：锁超时
* **答案**：

  * 设置合理超时时间
  * 使用唯一 value 做锁标识
* **原理解析**：防止持锁 goroutine crash 导致永久占用
* **面试提示**：生产经验题

**19. 如何实现可重入锁？**

* **考点**：高级锁设计
* **答案**：

  * value 中存 goroutine id + count
  * 同一个 goroutine 可重复加锁
* **原理解析**：保证同一客户端可重入
* **面试提示**：复杂锁设计

**20. 锁竞争高如何优化？**

* **考点**：性能优化
* **答案**：

  * 尽量减小锁粒度
  * 使用 try-lock 或随机重试
* **原理解析**：减少等待时间，提高吞吐量
* **面试提示**：生产场景优化

**21. 如何用 Lua 脚本保证锁释放安全？**

* **考点**：原子操作
* **答案**：

```lua
if redis.call("get",KEYS[1]) == ARGV[1] then
    return redis.call("del",KEYS[1])
else
    return 0
end
```

* **原理解析**：释放锁检查持有者，保证原子性
* **面试提示**：防止误删别人锁

**22. 高并发分布式锁可能出现什么问题？**

* **考点**：锁安全
* **答案**：

  * 超时释放太早 → 数据不一致
  * 节点网络分区 → 多个客户端持锁
* **原理解析**：分布式系统一致性问题
* **面试提示**：CAP 理解

## **四、限流（题 23–27）**

**23. 如何用 Redis 实现简单的固定窗口限流？**

* **考点**：高并发限流
* **答案**：

  * 使用 key+时间戳记录请求数
  * 例如每秒计数，超过阈值拒绝
  * 代码示例：

```lua
local key = KEYS[1]
local limit = tonumber(ARGV[1])
local count = redis.call("incr", key)
if count == 1 then
    redis.call("expire", key, 1)
end
if count > limit then
    return 0
end
return 1
```

* **原理解析**：通过 incr + expire 实现原子计数
* **面试提示**：考察限流实现能力

**24. 什么是滑动窗口限流？如何实现？**

* **考点**：精细化限流
* **答案**：

  * 用 ZSet 存请求时间戳
  * 每次请求清理过期时间戳，统计最近窗口内请求数
* **原理解析**：保证任意时间段内请求均匀分布
* **面试提示**：限流精度优化

**25. 如何用 Redis 实现漏桶限流？**

* **考点**：平滑流量
* **答案**：

  * 定义漏桶容量与流出速率
  * 请求进桶，按速率出桶
  * 可用 Lua 脚本实现原子操作
* **原理解析**：防止突发请求冲击系统
* **面试提示**：高并发系统稳定性

**26. 滑动窗口 vs 漏桶 vs Token Bucket 有何区别？**

* **考点**：限流算法理解
* **答案**：

  * 滑动窗口：统计固定时间窗口内请求数
  * 漏桶：平滑请求速率
  * Token Bucket：允许短时间突发 + 平均速率限制
* **原理解析**：不同策略适用于不同场景
* **面试提示**：算法理解能力

**27. 高并发限流如何避免 Redis 成为瓶颈？**

* **考点**：性能优化
* **答案**：

  * 本地计数 + 异步写入 Redis
  * 分片限流 key
  * pipeline 批量操作
* **原理解析**：减少 Redis 操作频率，提高吞吐量
* **面试提示**：系统性能优化

## **五、高并发优化（题 28–33）**

**28. Redis 大 key 如何处理？**

* **考点**：性能优化
* **答案**：

  * 避免一次操作大 key
  * 分片存储或批量读取
  * 使用 SCAN/LIMIT 遍历
* **原理解析**：单线程阻塞会影响吞吐
* **面试提示**：高并发场景优化

**29. Redis 热点 key 如何分散？**

* **考点**：热点问题
* **答案**：

  * 加前缀随机化 key
  * 分片存储到多个 hash 或 zset
* **原理解析**：避免单个 key 成为瓶颈
* **面试提示**：高并发热点处理能力

**30. Redis pipeline 与事务（MULTI/EXEC）区别？**

* **考点**：批量操作
* **答案**：

  * pipeline：一次发送多条命令，无事务保证
  * MULTI/EXEC：事务执行，保证原子性
* **原理解析**：pipeline 提高吞吐，事务保证一致性
* **面试提示**：吞吐量 vs 数据一致性理解

**31. Redis Lua 脚本的高并发优势？**

* **考点**：原子操作
* **答案**：

  * Lua 脚本原子执行
  * 避免多命令组合导致竞争
* **原理解析**：减少客户端往返、保证原子性
* **面试提示**：实战操作优化

**32. Redis 内存优化方法？**

* **考点**：缓存性能
* **答案**：

  * 小 hash 使用 ziplist
  * 避免大 key
  * 使用过期策略、LRU/LFU
* **原理解析**：减少内存占用，提升吞吐
* **面试提示**：生产经验题

**33. Redis 集群如何支持高并发？**

* **考点**：分布式架构
* **答案**：

  * 分片 sharding，跨节点存储数据
  * Client 根据 key 哈希选择节点
  * 主从复制 + 哨兵保证高可用
* **原理解析**：横向扩展，提高并发能力
* **面试提示**：分布式 Redis 理解
